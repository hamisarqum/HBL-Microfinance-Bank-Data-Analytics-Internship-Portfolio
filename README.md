# HBL Microfinance Bank: Data Analytics & Automation Portfolio 🏦

### 📌 Project Overview
This repository showcases the technical solutions and analytical workflows developed during my Data Analytics internship at **HBL Microfinance Bank** (July 2025 – September 2025). The core focus of this role was leveraging **Python** and **SQL** to automate high-volume regulatory data processing and optimize financial recovery performance.

---

### 🛠️ Key Technical Achievements
* **Regulatory Data Automation:** Engineered a Python-based validation tool to process **1.6 million State Bank of Pakistan (SBP) records**, successfully reducing manual processing time from hours to mere minutes.
* **Loan Recovery Optimization:** Utilized advanced SQL queries to analyze and optimize a dataset of **720,000 loan recovery records**, providing actionable performance insights.
* **Compliance & Error Tracking:** Designed and deployed **VBA-powered macros** and interactive dashboards to monitor and mitigate regulatory reporting errors.

---

### 📂 Repository Structure
* **`Latest cleaning tool by Hamis Arqum.py`**: The core Python script used for bulk record validation, cleaning, and workflow automation.
* **`Script.sql` through `Script-9.sql`**: A comprehensive collection of SQL scripts used for data optimization and financial performance reporting.
* **`Automating-Compliance-Data-Validation.pptx`**: Documentation detailing the logic, strategy, and impact of the automated compliance workflows.
* **`Loan recovery Analysis.docx`**: A detailed report outlining the findings and statistical interpretations from the large-scale recovery optimization project.

---

### 📈 Impact Summary
* **Scalability:** Successfully managed and analyzed datasets exceeding **1.6 million rows**.
* **Efficiency:** Achieved a significant reduction in manual labor by transitioning from legacy processes to automated Python scripts.
* **Accuracy:** Enhanced the reliability of regulatory data validation via automated error-tracking dashboards.

**Confidentiality Note:** To comply with banking privacy standards and institutional policies, all scripts and documents in this repository use synthetic or anonymized data structures that mirror real-world banking logic without exposing sensitive information.
